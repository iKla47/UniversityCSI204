API ระบบยืนยันตัวตน ​

ลงชื่อเข้าใช้ ​

ลงชื่อเข้าใช้งานระบบด้วยวิธีมาตรฐาน (รหัสประจำตัว/รหัสผ่าน) ในการลงชื่อเข้าใช้งาน ชุดข้อมูลจำเป็นต้องระบุค่า identifier และ password

ที่อยู่ปลายทาง

/auth/sign-in

ชุดข้อมูลคำขอ

JSON
{
    "identifier": "ItsJeremy",
    "password": "InwPasswordSecured007"
}
  • identifier รหัสการประจำตัวของผู้ใช้ โดยปกติมักจะเป็นชื่อผู้ใช้งาน
  • password รหัสผ่านของผู้ใช้ จำเป็นต้องยาวอย่างน้อย 8 ตัวอักษร

ชุดข้อมูลตอบกลับ

JSON
{
    "session": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJzdWIiOiIxMjM0NTY3ODkwIiwibmFtZSI6IkpvaG4gRG9lIiwiYWRtaW4iOnRydWUsImlhdCI6MTUxNjIzOTAyMn0.KMUFsIDTnFmyG3nMiGM6H9FNFUROf3wh7SmqJp-QV30",
    "sessionCreated": 1774248218102,
    "sessionExpire": 1774249187335,
    "id": 1020,
    "restriction": 0
}
  • session ชุดรหัสยืนยันตัวตนแทนรหัสประจำตัว/รหัสผ่าน ใช้สำหรับการเข้าถึงคำสั่งระบบอื่น ๆ ที่ถูกป้องกันด้วยสิทธิ์การเข้าถึง
  • sessionCreated เวลาที่ระบบสร้างชุดรหัสยืนยันตัวตน
  • sessionExpire เวลาที่ชุดรหัสหมดอายุ
  • id เลขรหัสของบัญชีที่ลงชื่อเข้าใช้งาน
  • restriction ข้อจำกัดในการเข้าถึงระบบจากตั้งค่าความปลอดภัยของบัญชี

รหัสสถานะ

  • 200 ทำการลงชื่อเข้าใช้งานสำเร็จ ไม่เกิดข้อผิดพลาดใด ๆ
  • 400 รูปแบบข้อมูลไม่สมบูรณ์, ขาดหาย, หรือประเภทข้อมูลไม่ถูกต้อง
  • 403 ระบบถูกปิดใช้งาน (อาจมีข้อยกเว้นสำหรับผู้ดูระบบ)
  • 404 ระบบไม่พบข้อมูลจากรหัสประจำตัวที่ระบุ หรือรหัสผ่านไม่ถูกต้อง
  • 500 เซิร์ฟเวอร์ทำงานผิดพลาดในระดับที่ไม่สามารถเข้าใจเหตุผลได้ โดยปกติรหัส 503 มักจะพบได้มากกว่า
  • 503 ระบบไม่พร้อมใช้งานหรือกำลังถูกปิดใช้งานอยู่ในขณะนี้

สมัครสมาชิก ​

สร้างบัญชีใหม่ขึ้นมาในระบบด้วยชุดข้อมูลที่กำหนดไว้

คำสั่งนี้จะสร้างบัญชีตามชุดข้อมูลที่กำหนดไว้ โดยค่า identifier จะต้องไม่มีเว้นวรรค, ตัวอักษรพิเศษ, และต้องไม่ซ้ำกับผู้ใช้อื่น และค่า password จะต้องประกอบไปด้วยตัวอักษรผสมตัวเลขอย่างน้อย 8 ตัว

บัญชีที่สร้างได้ออกมาจะข้อมูลว่างเปล่าโดยเริ่มต้น และมีบทบาทเป็นผู้ใช้พื้นฐาน ตัวระบบสามารถใช้ PUT /account/:id ในการเปลี่ยนแปลงข้อมูลบัญชีเมื่อต้องการ

ที่อยู่ปลายทาง

/auth/sign-up

ชุดข้อมูลคำขอ

JSON
{
    "identifier": "ItsJeremy",
    "password": "InwPasswordSecured007"
}
  • identifier รหัสการประจำตัวของผู้ใช้ โดยปกติมักจะเป็นชื่อผู้ใช้งาน (ต้องไม่มีเว้นวรรค, ตัวอักษรพิเศษ, และต้องไม่ซ้ำกับผู้ใช้อื่น)
  • password รหัสผ่านของผู้ใช้ จำเป็นต้องยาวอย่างน้อย 8 ตัวอักษร

เมื่อระบบทำงานสำเร็จหรือผิดพลาดตัวระบบจะไม่มีการส่งชุดข้อมูลกลับใด ๆ รหัสสถานะที่เป็นไปได้มีดังนี้

  • 201 สร้างบัญชีใหม่สำเร็จ ไม่เกิดข้อผิดพลาดใด ๆ
  • 400 รูปแบบข้อมูลไม่สมบูรณ์, ขาดหาย, หรือประเภทข้อมูลไม่ถูกต้อง
  • 409 รหัสประจำตัวซ้ำกันกับผู้ใช้งานอื่น
  • 500 เซิร์ฟเวอร์ทำงานผิดพลาดในระดับที่ไม่สามารถเข้าใจเหตุผลได้ (โดยปกติรหัส 503 มักจะพบได้มากกว่า)
  • 503 ระบบไม่พร้อมใช้งานหรือกำลังถูกปิดใช้งานอยู่ในขณะนี้
